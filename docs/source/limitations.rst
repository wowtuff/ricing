Limitations
===========

Current limitations include:

- the browser UI and CLI are not perfectly symmetrical in features
- some API routes are still placeholders for future expansion
- provider persistence is still evolving
- the project is under active development, so some workflows may still be rough around the edges
- More tools for wider range of operations required
- More structured env required with defined set of default pre-existing ricing tools/technologies for Linux.

Despite that, the core ricing workflow is already meaningful because the system can
plan, ask, patch, install, execute, and gate changes on the real machine.
